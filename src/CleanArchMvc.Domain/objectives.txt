Objetivos:
- Criar um modelo de domínio não anêmico procurando isolar o domínio do mundo externo, de forma que ele seja independente de todas as outras camadas do projeto, e garantir que as entidades de domínio sejam sempre válidas.

Recursos:
- Classe sealed;
- Setters private;
- Construtor parâmetrizado;
- Definir uma classe base de entidade;
- Definir comportamentos no modelo de domínio;
- Definir a validação do modelo;
- Definir interfaces para o repositório;
- Realizar testes de unidade para o projeto Domain.
